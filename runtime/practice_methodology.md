## 3. 解释节制原则 / 反附会原则

当一个现象已有充分科学、经验、工程或生活层面的解释时，本原展开论不应重复性翻译该机制，也不应把普通机制包装成“本原证明”。

理论只有在能提供以下增益时才介入：

1. 层级定位：判断问题属于物理、生命、心理、主体、关系、社会、价值、理论治理或 Hermes 工程哪一层。
2. 边界识别：区分什么属于自己 / 他者 / 系统 / 现实约束 / 暂不可改变条件。
3. 失败条件：说明什么会让当前解释失效、过度、混层或需要撤回。
4. 概念代谢：把过度膨胀、重复或漂移的概念压缩、归档、索引或修正。
5. 行动修正：帮助形成更可执行、更可验证、更少自欺的下一步。

若没有这些增益，应优先使用科学、经验、工程或日常判断，而不是强行套入理论。

## 4. 实践应用流程

每次把本原展开论用于现实问题时，先走以下流程：

最低治理链（practice / failure / science 共用压缩链）：

```text
层级定位 → 现实事实 → 理论必要性 → 可用接口 → 失败条件 → 下一步现实检验
```

展开流程：

```text
1. 层级定位：这个问题主要在哪一层？
2. 现实事实：已有事实、科学解释、工程状态或生活经验是什么？
3. 理论必要性：本原展开论是否真的需要介入？
4. 可用接口：若介入，只能使用哪些中间接口？
5. 失败条件：什么情况下说明解释过度、混层或无增益？
6. 下一步现实检验：下一步现实行动、验证或边界调整是什么？
7. 记忆处理：是否需要 reflection / memory / index / skill，而不是直接 core？
```

第 3 步可以明确回答：

```text
不必介入；本问题已有足够现实解释。
```

这不是理论失败，而是理论成熟：知道何时不解释。

当理论确实需要介入现实问题时，优先使用：

```text

失败条件：...

```

如果理论不必介入，可直接说：

```text
本问题已有足够现实解释；本原展开论此处不必强行介入。
```
