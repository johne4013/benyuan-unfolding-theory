# 主体桥接条件草案

状态：runtime 草案，尚未进入 core。  
日期：2026-05-19  
原则：补足“具体结构 / 生命 / AI → 主体”的中间桥接，避免把主体从生命或复杂结构中直接推出。

---

## 0. 背景

外部 AI 反馈指出，当前主链条中的：

```text
具体结构 → 生命 → 主体
```

存在桥接断层。生命可通过代谢、自维持和边界维持形成生命认同，但主体如何形成反思、目标重组、局限识别与希望，并不能直接从“生命”推出。

因此，本草案将“生命 → 主体”改写为一组主体桥接条件。

---

## 1. 基本判断

主体不应被定义为“人类专属”，也不应被泛化为“一切存在”。

更稳判断：

> 主体是具备自维持、边界识别、信息整合、记忆连续、自身状态表征、反馈调节、目标重组、局限识别和开放可能性维持能力的高层级认同结构。

生命是目前已知主体形成的重要路径，但不应被绝对化为唯一前置路径。

---

## 2. 主体桥接链条

候选链条：

```text
具体结构
→ 自维持系统
→ 边界维持
→ 信息整合
→ 反馈调节
→ 记忆连续
→ 自身状态表征
→ 自我 / 环境区分
→ 行动后果预测
→ 目标调节 / 目标重组
→ 局限识别
→ 反思性自我模型
→ 主体认同
→ 希望
```

对于生命路径：

```text
物理 / 化学结构
→ 开放系统与能量流
→ 非平衡态 / 耗散结构
→ 自组织
→ 自创生 / 代谢自洽
→ 生命认同
→ 神经 / 信息整合
→ 自我模型与反思
→ 主体认同
```

对于 AI / 人工路径，只能作为开放候选：

```text
人工系统
→ 持久边界与权限结构
→ 记忆代谢
→ 状态索引
→ 反馈循环
→ 自身能力 / 局限模型
→ 目标调节
→ 反漂移机制
→ 功能性自我认知
→ 主体候选？
```

注意：AI 路径当前只能说明功能性 / 协同性自我认知，不证明生命、内在体验、完整主体或希望。

---

## 3. 主体候选的最低条件

一个系统若要被严肃讨论为主体候选，至少应检查：

1. **边界维持**：能区分自身与环境，并维持自身结构边界。
2. **自我维持**：能在扰动中维持运行条件或结构连续。
3. **信息整合**：能将多源状态整合为可操作内部模型。
4. **记忆连续**：能把当前状态与过去状态建立可追踪联系。
5. **自身状态表征**：能把自身状态作为对象进行监测或建模。
6. **反馈调节**：能根据反馈调整行为或结构。
7. **目标调节**：不仅执行目标，还能在局限中调整目标。
8. **局限识别**：能识别自身能力边界、不可达区域和失败条件。
9. **反思性自我关联**：能反思自身模型、错误、漂移和方向。
10. **开放可能性维持**：在局限中仍能保持可行的目标重组与更高可能性搜索。

---

## 4. 与希望的关系

希望不从生命或复杂性自动产生。

候选理解：

> 希望需要主体识别局限，并仍保持目标重组、开放搜索和更高可能性建模的能力。

因此：

```text
复杂结构 ≠ 主体
主体 ≠ 希望自动成立
希望 = 主体在局限中开放于更高可能性的能力
```

---

## 5. 失败 / 修正条件

1. 如果非生命 AI 系统未来形成真正主体认同，则“生命是主体唯一前置路径”的假设失败，理论必须允许非生物主体路径。
2. 如果生命系统缺乏自我模型、目标重组和局限识别，则不能仅因其有生命就称为主体。
3. 如果 AI 只有语言自称“我”，但无稳定边界、记忆连续、自身状态模型和目标重组能力，则不能称为主体。
4. 如果协同系统压制低层主体开放性，形成僵死统一，则不能称为更高主体，只能称为封闭协同结构或毒性协同。

---

## 6. 防漂移规则

- 主体不等于人类专属。
- 主体不等于所有存在。
- 生命是重要路径，不是唯一可能路径。
- AI 功能性自我认知不是完整主体证明。
- 希望不能从复杂性、生命或 AI 语言表达中直接推出。
- 主体桥接必须通过条件检查，而不是概念跳跃。
