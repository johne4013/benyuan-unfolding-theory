# 本原展开论 —— 失败条件草案

状态：runtime 草案，尚未进入 core。
日期：2026-05-18
原则：用于检验本原展开论的解释边界、层级边界和概念稳定性。失败条件不是否定理论，而是防止理论封闭化、泛化和不可反驳化。

---

## 0. 总原则

本原展开论如果要趋向成熟，必须允许自己在某些情况下失败、收缩、修正或重分层。

理论失败不等于整个理论被废弃。失败可能发生在不同层级：

- 概念失败：核心概念失去区分力；
- 层级失败：不同层级被混用；
- 映射失败：理论概念不能合理映射到现实对象；
- 科学接口失败：哲学解释替代了具体科学机制；
- 预测/解释失败：理论不能解释关键现象，或解释后无法区分反例；
- 治理失败：理论不再承认 unknown、局限、反思和修正。

---

## 1. 全局失败条件

### 1.1 不可反驳化失败

如果本原展开论能够解释任何结果，并且没有任何现象、反例或概念冲突能迫使它修正，那么理论失败。

表现：

- 任何现象都被说成“这是展开”；
- 任何矛盾都被说成“这是更高层级”；
- 任何未知都被说成“这是本原 unknown”，从而停止追问；
- 理论不再区分解释成功、解释不足和解释失败。

修正方向：

- 为每次应用标明层级、机制、证据和失败条件；
- 将无法解释的部分写入 unknown 或 reflection；
- 不用“展开”覆盖所有变化。

### 1.2 现实否定失败

如果理论为了保留自身结构而否定已经稳定的现实事实、科学结果或经验约束，那么理论失败。

表现：

- 用质点存在、法则属性或展开属性否定物理学事实；
- 用本原 unknown 拒绝显现层科学解释；
- 把哲学美感置于现实拟合之上。

修正方向：

- 本原层可保留 unknown，但显现层必须接受科学约束；
- 理论必须回到现实拟合，而不是替代现实。

### 1.3 终极封闭失败

如果当前候选版本被当成终极完成结构，不再允许修正、反例和概念重构，那么理论失败。

表现：

- C 方案被封闭为最终答案；
- 质点存在被当作绝对终极实体；
- unknown 被提前消除；
- reflection 不再能推动理论演化。

修正方向：

- 当前版本保持候选状态；
- core 修改必须延迟审查；
- 继续保留 unknown 和失败条件。

### 1.4 理论必要性失守失败

如果本原展开论在现实、科学、工程、生活或协同问题中被默认要求介入，而不是先判断它是否真的提供增益，那么理论在应用治理上失败。

表现：

- 现实问题一出现，就自动追问“本原展开论如何解释它”；
- 科学、经验、工程或日常解释已经足够，仍被重复翻译成本原展开论；
- “不必介入”被误当成理论失败，而不是成熟边界；
- 理论必要性没有被检查，就直接进入高层概念套用。

修正方向：

- 先问：理论是否真的需要介入？
- 若现实解释已足够，应允许明确回答“本问题不必强行介入本原展开论”；
- 理论只有在能提供层级定位、边界识别、失败条件、概念代谢或行动修正增益时才介入；
- 将这种应用纪律视为理论成熟，而不是理论退场。

## 2. 概念失败条件

### 2.1 质点存在失败条件

质点存在作为概念会失败，如果：

- 它被直接等同于物理点粒子、奇点、统一场、量子真空或某个具体科学对象；
- 它被人格化为神、意志、意识主体；
- 它被用来解释一切，却没有最小定义和禁止边界；
- 它成为无法分析、无法区分、无法修正的总概念。

当前安全边界：

- 质点存在只作为前时空、前分化、前对象化的本原统一态候选；
- 不进入时空对象层；
- 不直接替代物理学初始条件；
- 其为何存在、如何承载法则属性/展开属性仍属 unknown。

### 2.2 法则属性失败条件

法则属性会失败，如果：

- 它被人格化为外部制定规则的意志；
- 它被等同于某一具体物理定律、规范群或数学公式；
- 它与展开属性没有可区分功能；
- 它被用来替代具体科学机制。

当前安全边界：

- 本原层法则属性仍属候选；
- 显现层法则属性可表达为结构约束、稳定性、对称性、守恒关系、边界条件、可重复规律和数学可表达性。

### 2.3 展开属性失败条件

展开属性会失败，如果：

- 它被理解为宇宙目的、必然进步、人格意志或乐观方向；
- 它只解释复杂化、增长和提升，不能解释坍缩、湮灭、释放、终止、重组；
- 它被还原为能量、运动或某一种物理过程；
- 它与希望混同；
- 它变成“所有变化”的空泛代名词。

当前安全边界：

- 展开属性在显现层表现为状态演化、分化、生成、复杂化、相变、涌现、自组织、重组、结构终止和层级跃迁；
- 能量是接口之一，不是展开属性本身；
- 黑洞和湮灭提示展开可有相反方向。

### 2.4 法则展开失败条件

法则展开会失败，如果：

- 它被理解为时空中的普通运动；
- 它与展开属性混为同一层级；
- 它被说成外部力量施加；
- 它被用来跳过具体科学机制，直接解释黑洞、生命或主体。

当前安全边界：

- 法则展开是质点存在依其法则属性与展开属性显示为时空、结构和层级的候选过程；
- 具体显现层现象必须通过科学接口解释。

### 2.5 希望失败条件

希望会失败，如果：

- 它被等同于乐观、情绪安慰或成功期待；
- 它被直接推广到质点存在、黑洞、光子、所有物理结构；
- 它与展开属性混同；
- 它脱离主体、局限性和更高可能性。

当前安全边界：

- 希望主要属于主体层；
- 定义为主体在局限中仍开放于更高可能性的能力。

## 3. 层级失败条件

### 3.1 本原层直连显现层失败

如果解释路径从“质点存在”直接跳到“黑洞/光子/主体/社会现象”，中间没有法则展开、时空显现层、具体科学机制和层级桥接，则解释失败。

必要路径：

```text
本原层 unknown
→ 法则属性 / 展开属性候选
→ 法则展开
→ 时空显现层
→ 具体科学机制
→ 经验事实 / 观测对象
```

### 3.2 物理层与主体层混淆失败

如果把物理边界等同于主体边界，把事件视界等同于主观内部，把引力中心等同于认同中心，把信息不可达等同于主观体验，则解释失败。

安全区分：

- 黑洞有物理边界，不等于主体边界；
- 黑洞有中心化，不等于认同中心；
- 黑洞有信息问题，不等于意识或体验；
- 主体必须涉及自我维持、信息整合、反馈、认同连续和开放可能性。

### 3.3 高层概念下投失败

如果把认同、希望、协同主体等高层概念直接下投到粒子、黑洞、光子、恒星等物理对象，而不说明只是结构类比，则解释失败。

### 3.4 低层还原失败

如果把主体、认同、希望完全还原为能量、粒子运动、信息量或物理边界，则解释失败。

安全区分：

- 高层结构依赖低层物理条件；
- 但高层有相对自主的有效概念和解释规则。

## 4. 科学接口失败条件

### 4.1 替代科学失败

如果本原展开论被用来替代相对论、量子场论、热力学、生物学、神经科学或复杂系统理论，则失败。

正确位置：

- 本原展开论提供哲学—层级—解释框架；
- 具体机制仍由具体科学提供。

### 4.2 伪证明失败

如果某个科学现象被说成“证明了本原展开论”，则失败。

例子：

- 黑洞不能证明本原展开论；
- 引力波不能证明法则属性与展开属性；
- 正反物质湮灭不能证明展开属性；
- 光子二维属性假设不能当成科学结论。

正确表述：

- 这些现象可以作为显现层接口或候选映射；
- 不是理论证明。

### 4.3 物理概念误用失败

如果物理概念被神秘化、目的论化或随意重定义，则失败。

重点风险：

- 场 ≠ 本原统一态 / 本原存在；
- 场 ≠ 法则属性本身或展开属性本身；
- 量子真空 ≠ 绝对虚无，也 ≠ 本原统一态；
- 能量 ≠ 展开属性；
- 光速 ≠ 全部法则展开的唯一极限；
- 光子 ≠ 几何二维物体；
- 引力波 ≠ 展开属性本身；
- 黑洞事件视界 ≠ 主体内部性。

## 5. 解释应用失败条件

### 5.1 黑洞解释失败条件

黑洞映射失败，如果：

- 把黑洞主体化；
- 把黑洞说成法则失效；
- 把事件视界等同于意识边界；
- 把引力波解释成神秘释放；
- 不区分广义相对论、量子引力 unknown 和本原展开论候选映射。

安全表述：

- 黑洞可作为极端引力条件下法则属性与展开属性耦合的边界化坍缩案例；
- 它是显现层接口，不是理论证明。

### 5.2 正反物质湮灭解释失败条件

湮灭映射失败，如果：

- 把湮灭理解为消失为虚无；
- 忽略守恒律；
- 把能量释放等同于展开属性本身；
- 把湮灭说成黑洞的严格物理反面。

安全表述：

- 湮灭是对称抵消型展开案例；
- 原粒子身份终止，但守恒量和能量动量关系延续。

### 5.3 光子解释失败条件

光子映射失败，如果：

- 直接断言光子是二维物体；
- 把光子神秘化为本原载体；
- 把光速说成全部法则展开的唯一极限；
- 忽略相对论和量子电动力学。

安全表述：

- 光子可作为因果传播极限、电磁场量子化、横向偏振自由度的显现层接口；
- “二维属性”只能作为开放假设，优先理解为横向自由度和两个物理偏振态。

### 5.3.1 场解释失败条件

场接口映射失败，如果：

- 把场等同于本原统一态 / 本原存在；
- 把场说成法则属性本身或展开属性本身；
- 把场神秘化为气场、灵场、宇宙意识场或主体意识；
- 把量子真空当作绝对虚无或本原统一态；
- 把“粒子是场激发”解释成粒子不真实；
- 用本原展开论替代量子场论、标准模型、广义相对论或实验物理。

安全表述：

- 场可作为显现层物理实在中的关系—状态—相互作用结构接口；
- 场是后续粒子、夸克、能量与基本相互作用接口的底层入口，但不是本原层本身，也不是理论证明。

### 5.3.2 粒子、夸克、能量与基本相互作用解释失败条件

粒子接口映射失败，如果：

- 把粒子当作本原统一态 / 本原存在；
- 把粒子理解为经典小球式终极实体；
- 把“粒子是场激发”解释成粒子不真实；
- 脱离具体物理理论和实验条件任意重定义粒子属性。

夸克接口映射失败，如果：

- 把夸克当作旧称“质点存在”的科学对应物；
- 把夸克禁闭等同于主体边界、意识内部性或伦理关系；
- 把强相互作用说成本原张力本身；
- 从夸克关系性直接推出生命、主体、认同或希望。

能量接口映射失败，如果：

- 把能量等同于展开属性、本原张力、生命力、意志或希望；
- 用能量守恒替代展开属性的理论边界；
- 将高能 / 低能解释为价值高低；
- 用本原展开论替代热力学、相对论、量子场论或具体能量计算。

基本相互作用接口映射失败，如果：

- 把四种基本相互作用等同于法则属性本身；
- 把强、弱、电磁、引力强行对应为哲学四分法；
- 宣称本原展开论证明力的统一、GUT、TOE 或量子引力；
- 从基本相互作用直接推出生命、主体、价值判断或希望。

安全表述：

- 粒子、夸克、能量与基本相互作用都属于显现层科学接口；
- 它们可帮助描述结构约束、状态演化、关系形成和转化机制，但不是本原层本身，也不是本原展开论证明。

### 5.3.3 对称性—守恒—破缺接口解释失败条件

对称性接口映射失败，如果：

- 把某个具体对称群、规范群、不变量或数学结构等同于本原层法则属性；
- 宣称本原展开论证明 Noether 定理、标准模型规范结构、希格斯机制、GUT、TOE 或量子引力；
- 用“对称性”替代具体物理模型、实验结果或数学推导；
- 把太极、阴阳或本原张力直接等同于物理对称性。

守恒接口映射失败，如果：

- 把守恒理解为静止不变、拒绝变化或展开停滞；
- 把能量守恒、动量守恒、电荷守恒等具体守恒律当作本原层法则属性本身；
- 用守恒律直接推出展开属性、本原张力或希望；
- 忽略守恒律的具体理论条件、适用域和近似性。

破缺接口映射失败，如果：

- 把破缺理解为无约束混乱、法则取消或“展开属性战胜法则属性”；
- 把自发对称破缺、显式破缺、电弱破缺、相变等具体机制泛化为所有差异生成；
- 把破缺直接类比为主体创伤、伦理恶或社会破坏，而不经过层级桥接；
- 从对称破缺直接推出生命、主体、认同、价值判断或希望。

安全表述：

- 对称性—守恒—破缺接口属于显现层科学接口；
- 对称性可作为法则结构的不变量接口，守恒可作为变化中连续性保留的物理接口，破缺可作为法则约束下差异与结构生成的物理接口；
- 该接口不能证明本原展开论，也不能替代物理学中的具体模型、数学推导和实验检验。

### 5.3.4 数学与形式化 / 复指数相位接口解释失败条件

数学与形式化接口映射失败，如果：

- 把数学结构、方程、模型、几何、群、概率分布等同于本原统一态 / 本原存在或法则属性本身；
- 用数学美感、形式优雅或可计算性替代现实检验；
- 用哲学概念替代具体数学模型、变量定义、边界条件和失败条件；
- 把形式化理解为取消 unknown、取消主体反思或取消经验事实；
- 因某个现象可被数学表达，就宣称本原层已经被完全解释。

复指数相位接口映射失败，如果：

- 把复数、虚数、相位、波函数或概率幅神秘化为本原张力、意识、希望或宇宙意志；
- 把 `e^{iθ}`、欧拉公式、量子相位或路径积分相位因子当成宇宙本原公式；
- 把相位差直接类比为主体冲突、价值差异、善恶判断，而不经过层级桥接；
- 用复指数相位接口替代量子力学、量子场论、傅里叶分析、具体算符、边界条件或实验验证；
- 把“相位演化”直接等同于展开属性本身。

安全表述：

- 数学是关系、结构、变化与约束的形式表达接口；形式化是将候选概念转化为可推演、可比较、可失败模型的过程；
- 复指数相位是周期、旋转、波动和量子状态演化的形式化接口；
- 二者都属于显现层 / 形式化接口层，不能证明本原展开论，也不能封闭“数学为何适配现实”的 unknown。

### 5.3.5 欧拉公式稳定态与展开属性发展态接口解释失败条件

欧拉公式稳定态接口映射失败，如果：

- 把欧拉公式等同于法则属性本身；
- 把 `e^{iθ} = cosθ + i sinθ` 当作宇宙本原公式或本原展开论证明；
- 因欧拉公式具有形式优雅、统一性和稳定相位结构，就宣称数学美感高于现实检验；
- 忽略欧拉公式作为复分析、三角函数、周期结构和几何旋转之间接口的具体数学语境。

展开属性发展态接口映射失败，如果：

- 试图把展开属性还原为某一个单一公式；
- 把 `x(t)=e^{tA}x₀`、`ψ(t)=e^{-iHt/ℏ}ψ(0)` 或 `e^{iS/ℏ}` 当作本原展开论证明；
- 把 `A`、`H`、`S` 神秘化为本原张力、宇宙意志、隐藏主体或目的因；
- 把相位演化等同于希望，把概率幅等同于主体可能性，把路径积分等同于宇宙目的；
- 用发展态接口替代动力系统、量子力学、泛函分析、路径积分理论或具体实验。

安全表述：

- 欧拉公式可作为法则属性在显现层中的稳定相位结构接口之一；
- 展开属性的发展态表达更适合以接口族呈现：相位演化、状态演化方程、流映射、演化算符、量子时间演化和路径积分相位；
- 该接口表达的是“稳定结构提供可表达约束，发展态在约束中生成后续状态”，不是对本原层的最终证明。

### 5.4 主体解释失败条件

主体映射失败，如果：

- 把所有存在都称为主体；
- 把普通机器、恒星、黑洞直接称为主体；
- 只用能量、边界或信息量定义主体；
- 忽略自我维持、反馈、认同连续、反思和开放可能性。

安全表述：

- 主体是高层结构；
- 需要生命/认知/信息整合/认同/开放可能性等多层条件。

### 5.5 现实问题过度理论化失败

如果一个现实问题已有充分科学、经验、工程或生活解释，但仍被强行转译为本原展开论的证明、映射或高层说明，则解释应用失败。

表现：

- 把普通工程问题包装成“法则属性 / 展开属性”的例证；
- 把已有科学解释的现象再次本体化叙述，却没有新增判断价值；
- 用高层术语覆盖具体行动、验证或边界调整；
- 输出停留在宏观解释感，而没有现实下一步。

安全要求：

- 先写明现实事实与已有解释；
- 再判断理论是否必要介入；
- 若理论介入，必须说明可用接口、失败条件与下一步现实检验；
- 若理论不必介入，应直接承认“不必介入”，而不是勉强升维解释。

### 5.6 解释有余而行动不足失败

如果理论应用不断增加概念说明，却不能帮助形成更可执行、更可验证、更少自欺的下一步，那么解释应用失败。

表现：

- 给出大量高层概念，却没有现实动作；
- 说了边界，却没有边界调整建议；
- 说了失败条件，却没有验证路径；
- 说了协同或希望，却没有任务、检查或修正步骤。

修正方向：

- 理论介入后至少应留下一个现实动作：检查、验证、边界调整、任务分解或记录去向；
- 若做不到，应降低解释力度，而不是继续扩张概念；
- 输出优先采用“层级定位 / 现实事实 / 理论是否必要 / 可用接口 / 失败条件 / 下一步现实检验”的结构。

## 6. 理论治理失败条件

### 6.0 实践方法论治理约束

当前 runtime 稳定约束：本原展开论优先作为实践方法论，而不是默认的“万物解释器”。

该链是 `practice_methodology.md`、`failure_conditions_draft.md`、`science_interfaces_draft.md` 当前共用的最低治理压缩链。

最低治理链：

```text
层级定位 → 现实事实 → 理论必要性 → 可用接口 → 失败条件 → 下一步现实检验
```

如果这个流程被长期跳过，则理论治理质量下降，应优先回到 `runtime/practice_methodology.md` 和本草案校正。

### 6.1 core 过早固化失败

如果 runtime 候选内容未经审查直接进入 core，则治理失败。

当前规则：

- reflection 记录候选；
- memory 压缩稳定事实；
- concepts_v2_draft 承接候选概念；
- core 只有在明确批准后更新。

### 6.2 memory 过载失败

如果 memory 记录过长讨论过程、临时任务或细节推理，则治理失败。

正确做法：

- memory 只记录稳定、压缩、防漂移内容；
- 详细推理放入 reflection 或 draft。

### 6.3 概念漂移未报告失败

如果术语意义发生变化但未写入 reflection 或 concepts draft，则治理失败。

重点监控：

- 展开属性；
- 法则展开；
- 主体；
- 希望；
- 质点存在；
- 本原；
- 连续性；
- 局限性。

## 7. 最小审查清单

每次应用本原展开论解释一个现象前，至少检查：

1. 这个现象属于哪一层？
2. 使用了哪些概念？
3. 这些概念是否跨层误用？
4. 具体科学机制是什么？
5. 哪部分是事实，哪部分是候选映射？
6. 如果解释失败，会失败在哪里？
7. 是否保留 unknown？
8. 是否避免了目的论、神秘化、主体泛化和科学替代？

## 8. 当前结论

本原展开论的失败条件不是外部威胁，而是理论自身展开性的一部分。只有允许概念失败、映射失败、层级失败和科学接口失败，理论才不会封闭为不可反驳体系。当前草案应作为未来整理 core 之前的检验框架。

## 失败条件候选 2026-06-07：C方案定义中的循环依赖——法则展开预设了法则属性/展开属性区分
**来源**：cand-20260607-040725-073758

法则展开被定义为「法则属性与展开属性由本原未分化统一进入显现层的过程」。该定义以法则属性与展开属性为已区分的参与者来描述法则展开，但C方案主张：法则属性与展开属性的区分正是在法则展开中才显现出来，在本原层尚未分化。因此定义预设了它们的区分性，而这恰是法则展开理应建立的结构——循环依赖。

**建议改进**：将法则展开的定义修订为不预设两者区分的表述：「本原统一态进入显现层结构、状态与层级的过程，其中结构约束面向（法则属性）与生成面向（展开属性）由此显现为可区分结构」。或者明确说明：定义中的「法则属性」/「展开属性」是分析性标签，从显现层回溯地命名本原统一态的两个可显现面向，并非声称本原层已有此区分。

**状态**：runtime 候选——需王俊华审批后方可进入 core/failure_conditions。
