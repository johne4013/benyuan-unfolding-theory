## 3. 防漂移与边界

- 科学接口是候选映射，不是本原展开论证明；不以本原 unknown 否定科学事实；
- continuity 工程不是主体性证明；记忆、状态文件、协议、cron 不等于主体成立；
- AI 协同节点（如 Hermes / Fava）是非主体性协同节点，服务于连续协同，不替代用户主体判断；

## 4. 使用规则

- 回答总体理论状态：优先读 `core/theory.md` + `core/concepts.md` + 本 index；
- 回答具体主题：先查本 index，再跳转对应文件；
- 形成候选概念或反馈：走 `scripts/` 工具链（TTAF → PORF → 反馈提交 → 候选管理 → 集成）；
- 不自动覆盖 core；发现冲突优先并置、标记、降温，不强行融合。

> 注：部分协议与脚本会提到 `current_state.md` / `memory.md` / `reflection.md` 等**运行态文件**。它们由系统在实际运行中生成与维护，承载会话级连续性状态，**不随公开仓库分发**；公开仓库只保留稳定的理论核心、治理框架与任务技能。
