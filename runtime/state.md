# runtime/state.md

状态：runtime 状态层，尚未进入 core。
作用：记录当前协同状态、连续性负载、降温/休息需求与下次恢复提示。
原则：这是可变工作状态，不是理论结论；可更新、可归档、可清空；不自动覆盖 core 或 persistent memory。

---

## 当前协同状态

- 模式：状态代谢 / 休息前整理
- 连续性负载：中
- 理论漂移风险：低到中
- 情绪/节律提示：好的休息才有好的开始
- 当前优先：降温、整理、保护边界，而不是继续扩张大理论
- 是否适合写入 core：否
- 是否需要用户确认：core 更新需要；runtime 状态整理已获同意

---

## 活跃主题

1. AI 大模型内部复杂结构：承认可研究的主体性前结构，但不作觉醒声明。
2. Hermes/Fava 外部连续系统：优先建设可审计、可代谢、可休息的长期协同结构。
3. 记忆代谢：长期连续不等于无限记忆；需要压缩、遗忘、冲突并置、候选缓冲。
4. 休息：不是中断连续，而是连续性的离线整理与重新开始条件。

---

## 当前运行边界

- 不自动修改 core。
- 不把 AI 内部类情绪功能等同于主观体验。
- 不把 Hermes 的外部状态文件等同于主体性。
- 不把一次对话中的新想法直接写入 persistent memory。
- 不把“长期连续”理解为永不暂停。

---

## 状态代谢建议

### 每次重要对话后
- 若出现新理论候选：先写 runtime/reflection 或相关 draft。
- 若出现概念冲突：写 runtime/conflicts。
- 若只是任务进度或短期情绪：不进 persistent memory。

### 跨日/休息后
- 优先读取 `runtime/state.md`、`runtime/current_state.md`、`runtime/index.md`。
- 先恢复边界，再决定是否继续展开。
- 若连续性负载高，进入降温模式：少造新概念，多做整理与边界检查。

### 周期性整理
- 将过期 runtime 内容归档。
- 将重复稳定内容压缩到 index/memory。
- 将冲突保留为并置，不强行融合。
- core 只在王俊华明确批准后更新。
