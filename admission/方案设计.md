# 心脏外科入院记录生成系统 —— 设计方案

## 一、定位

入院记录与出院告知是同一患者文书流程的两端，复用已验证的架构（单文件零依赖网页 +
知识库 + 浏览器打印转 PDF）。与出院告知不同，入院记录是**叙述性病历**，受《病历书写
基本规范》约束，须连续成文、项目完整。本工具核心：**用结构化的最小录入，自动生成一份
规范的连续文本病历草稿**，而非填表格。

- 形态：`admission/index.html` 单文件应用，与 `cardiac-discharge/` 并列，构成科室
  "入院—出院"文书小套件。零依赖、离线、`localStorage` 暂存、浏览器打印转 PDF。
- 左填右预览：左侧结构化录入，右侧实时生成规范病历正文（A4，可直接编辑、打印、转 PDF）。
- 固定抬头：汉中市中心医院 心脏外科。

## 二、内容模块（按病历书写规范）

| 模块 | 录入策略 |
|------|----------|
| 一般项目 | 表单字段，记录时间默认当前 |
| 主诉 | 症状 chips + 持续时间 → 自动成句，校验字数（建议≤20字） |
| 现病史 | 「按主诉生成模板」骨架 + 可点选片段 + 自由文本润色 |
| 既往史 | 勾选基础疾病（复用心外科分类）自动成句 + 手术/过敏/传染病默认表述 |
| 个人史/婚育史/家族史 | 默认模板 + 吸烟饮酒结构化；月经史按性别联动显示 |
| 体格检查 | **★ 默认正常值预填，医生只改异常**；生命体征结构化，心脏专科单列 |
| 辅助检查 | 自由文本（心电图/彩超/CT/化验） |
| 初步诊断 | 勾选心外科诊断分类自动编号 + 自定义追加行 |
| 签名 | 记录医师签名 / 日期栏 |

## 三、关键效率设计

1. **体格检查"默认正常、只改异常"**：预填整套标准阴性描述，医生只改有异常的系统，
   心脏专科给出可填空的结构化表述（心率__次/分、律齐、各瓣膜区杂音__级）。
2. **主诉/现病史半结构化**：主诉症状点选 + 时间自动成句并校验字数；现病史一键生成
   规范骨架句式，再自由润色。
3. **诊断与既往史复用出院系统分类**：高血压、冠心病、糖尿病、感染性心内膜炎、瓣膜病、
   先心病、主动脉夹层、心脏肿瘤、心包积液等三套文书术语一致。

## 四、合规与风险

- 遵循《病历书写基本规范》：入院记录须于入院 **24 小时内**完成；本工具仅生成**草稿**，
  必须由经治医师审核、修改并签字方为正式病历，页面已内置该声明。
- 内容模板须经科室审定。
- 患者隐私：纯本地运行，数据不出科室电脑；`localStorage` 暂存，提供一键清空。

## 五、实施分期

| 阶段 | 内容 |
|------|------|
| 一期（本次交付） | 单文件原型：全部模块 + 默认正常体检 + 主诉/现病史/诊断结构化 + 打印/PDF |
| 二期 | 与出院告知共享患者数据（同一患者键）；科室自定义体检模板 |
| 三期（可选） | 后端化 / 对接 HIS 自动带入患者信息 |
