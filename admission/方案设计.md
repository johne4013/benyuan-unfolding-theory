# 心脏外科 主诉/现病史/查体 快速生成 —— 设计方案

## 一、定位（v3）

本工具是**文本快速生成器**：医生勾选病变、点选要素，即时生成**主诉、现病史、查体、
初步诊断**四段文本，**复制粘贴到内部系统（HIS/电子病历）**使用。不录入任何患者身份
信息，不做页面打印/排版——输出就是可直接复制的文本框。

- 形态：`admission/index.html` 单文件零依赖网页，与 `cardiac-discharge/` 并列；
  离线可用、`localStorage` 暂存草稿。
- 每段输出带「复制」按钮，顶部「复制全部」一次复制四段；文本框均可手动修改。

## 二、病种细化 + 多选（核心机制）

病变细化到具体类型并可**多选**，勾选后主诉症状、现病史伴随症状、查体杂音/体征、
初步诊断均按所选病变**聚合/联合生成**：

| 分组 | 细化病变 |
|------|----------|
| 瓣膜病 | 二尖瓣狭窄、二尖瓣关闭不全、主动脉瓣狭窄、主动脉瓣关闭不全、三尖瓣关闭不全 |
| 冠心病 | 冠心病/心绞痛、陈旧性心梗 |
| 先天性心脏病 | 房间隔缺损、室间隔缺损、动脉导管未闭、法洛四联症 |
| 心衰/心律失常 | 心力衰竭、心房颤动 |
| 心包积液 | 心包积液 |
| 主动脉/血管 | 夹层 A 型、夹层 B 型、胸主动脉瘤、腹主动脉瘤 |
| 其他 | 感染性心内膜炎、心脏黏液瘤 |

每条病变在知识库中携带：主诉症状贡献、现病史伴随症状贡献、查体片段
（心 border/rhythm/murmur、肺、腹、下肢）、诊断；多选时自动去重合并。

## 三、四段生成逻辑

1. **主诉**：症状 chips（所选病变贡献 + 通用心血管症状，去重）点选拼接 + 持续时间
   → 自动成句，>20 字提示。
2. **现病史**：五组点选（诱因/起病方式单选；伴随症状[病变贡献+通用]、院外诊治、
   一般情况多选）→「组装现病史」按规范句式拼接成段，____ 占位提醒补全，可继续改。
3. **体格检查（心/肺/腹/下肢）**：生命体征 + 四系统。点「按所选病种联合生成查体」，
   多个瓣膜病变的**杂音自动合并**到心脏一栏，心律（如房颤）、心界自动调整；累及
   肺/腹/下肢的病变写入对应系统，未累及者保持正常默认；生成后可手动修改。
4. **初步诊断**：按所选病变自动编号列出，可手动增删。

## 四、合规说明

- 生成内容为**辅助草稿**，复制入内部系统后须经治医师核对、修改并签字；页面内置声明
  并提示核对 ____ 占位。
- 纯本地运行，患者数据不出科室电脑。
- 模板（症状、片段、体征、诊断措辞）须经科室审定，全部集中在文件顶部
  `TEMPLATES` / `COMMON` / `EXAM` 数据区，可自行增删。

## 五、迭代记录

- v1：完整入院记录生成（含一般项目、既往史、个人史等全部模块）。
- v2：重定位为快速采集器——四病种联动点选，输出复制到内部系统。
- v3（当前）：进一步聚焦——
  - 去除姓名等所有患者信息及打印/A4 预览，输出即可复制的文本框（主诉/现病史/查体/诊断各带复制按钮 + 复制全部）。
  - 重点优化主诉、现病史、主要查体（**仅心 / 肺 / 腹部 / 下肢**四系统）。
  - **病种细化到具体病变并可多选**：二尖瓣狭窄/关闭不全、主动脉瓣狭窄/关闭不全、三尖瓣关闭不全、房/室缺、动脉导管未闭、法洛四联症、夹层 A/B 型、心包积液、房颤、心衰、感染性心内膜炎、黏液瘤等共 20 条。
  - **查体按所选病变联合生成**：多个瓣膜病变的杂音、心律、心界自动合并到"心脏"一栏；累及肺/腹/下肢的病变（如心衰、三尖瓣关闭不全）自动写入对应系统，未累及者保持正常默认。
  - 移除 v2 中出故障的"点体征按钮写入"机制，改为确定性的"按病种一键生成查体"。
