# 心脏外科患者出院告知系统 —— 可行性方案

## 一、需求概述

为心脏外科出院患者生成个性化《出院告知单》，内容覆盖四大模块：

| 模块 | 内容 | 生成方式 |
|------|------|----------|
| 1. 一般注意事项 | 以心力衰竭管理为基底（限水、限盐、每日体重、活动耐量等）；伤口/穿刺、胸骨保护等手术相关内容在勾选手术后才追加显示 | 基底固定 + 手术联动 |
| 2. 内科合并症注意事项 | 高血压、冠心病、糖尿病、房颤、感染性心内膜炎、高脂血症、慢性肾病、COPD、脑卒中史等（心衰已并入基底） | 按勾选的诊断动态生成 |
| 3. 术后专项要求 | 搭桥、机械瓣/生物瓣置换、瓣膜成形、主动脉夹层开放手术、腔内隔绝术（TEVAR）、先心病矫治/封堵术、心脏肿瘤摘除、TAVR、心包穿刺 | 按勾选的手术类型动态生成 |
| 4. 出院用药医嘱 | 药品名称/剂量/用法/疗程表格 + 按药物类别自动汇总的用药注意事项；内置常用药品目录，点选即自动填入常见用法并关联注意事项，也支持手动录入 | 点选或手填，注意事项自动聚合去重 |

固定信息：抬头为汉中市中心医院心脏外科；随访电话默认 0916-2680021，复诊地点默认心脏外科二楼门诊。
输出要求：网页填写、A4 版式预览、一键打印、生成 PDF。

## 二、技术方案

### 2.1 总体架构：单文件纯前端应用（推荐，已实现）

- **形态**：一个自包含的 `index.html`（HTML + CSS + 原生 JavaScript），无任何外部依赖。
- **知识库**：一般注意事项、各疾病/各术式注意事项、各药物类别注意事项全部以结构化 JS 数据对象内置于页面中，便于科室自行维护扩充。
- **运行环境**：双击即可在任意现代浏览器打开；可放在医院内网静态服务器、共享盘、护士站电脑桌面，**无需后端、无需联网**——符合医院内网隔离的实际环境。
- **打印 / PDF**：使用浏览器原生打印能力 + `@media print` 专用版式（A4、隐藏表单区、仅输出告知单正文）。浏览器打印对话框中选择"另存为 PDF"即得 PDF 文件，零依赖且排版保真。
- **数据暂存**：使用 `localStorage` 自动保存草稿，刷新不丢失；不向任何服务器传输患者数据，规避隐私合规风险。

### 2.2 选型理由

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| **单文件纯前端**（本方案） | 零部署、零依赖、离线可用、数据不出本机 | 知识库更新需改文件 | ✅ 采用 |
| 前后端分离（如 Vue + 后端） | 知识库集中管理、可统计 | 需服务器、运维、等保合规 | 二期可演进 |
| 嵌入 HIS/电子病历 | 与医嘱系统打通自动带出诊断 | 需厂商接口配合，周期长 | 远期目标 |
| html2pdf.js / jsPDF 生成 PDF | 不弹打印框 | 中文字体需内嵌（体积大）、排版不如浏览器原生 | 不采用，浏览器打印转 PDF 已足够 |

### 2.3 页面结构

```
┌──────────────────────────────────────────────┐
│ 工具栏：生成告知单 / 打印 / 导出PDF / 清空    │
├──────────────────┬───────────────────────────┤
│ 左：医生填写区    │ 右：A4 实时预览区          │
│  ① 患者基本信息   │  医院抬头 + 患者信息表      │
│  ② 内科诊断勾选   │  一、一般注意事项           │
│  ③ 手术类型勾选   │  二、合并症注意事项（动态）  │
│  ④ 出院带药录入   │  三、术后专项要求（动态）    │
│  ⑤ 复诊安排      │  四、出院带药及用药注意       │
│                  │  五、紧急就医指征 + 签名栏    │
└──────────────────┴───────────────────────────┘
打印时仅输出右侧预览区（A4 版式）
```

### 2.4 知识库数据结构

```js
const KB = {
  general:  [ "……" ],                              // 一般注意事项
  redFlags: [ "……" ],                              // 紧急就医指征
  diseases: { ht: {label:"高血压", items:[…]}, … }, // 内科合并症
  surgeries:{ cabg:{label:"冠脉搭桥(CABG)", items:[…]}, … },
  drugClasses:{ warfarin:{label:"华法林", items:[…]}, … }
};
```

科室维护人员只需修改这一处数据即可增删条目，无需懂前端框架。

## 四、风险与合规说明

1. **医学内容仅为模板参考**，正式启用前必须由心脏外科主任/药剂科审定，页面已内置"以主管医师医嘱为准"免责声明及医患双签名栏。
2. **患者隐私**：纯本地运行，数据不出科室电脑；草稿存于浏览器本地，提供一键清空。
3. **浏览器兼容**：Chrome / Edge / Firefox 现代版本均可；医院如有老旧 IE 环境需提示更换浏览器。
